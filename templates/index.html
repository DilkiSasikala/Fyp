Digital Advertisement Click Fraud Detection

This application is a prototype for the click fraud detection of digital advertisements.

User Manual

Please read the user mannual on how to detect click fraud with this application

This applcation takes a .csv file as an input. This file must contain the click data of advertisements. The features that should be included in the file are ip, app, device, os, channel, hour and day. Once the file is uploaded the prediction process will initiate.
**Failing to input a file that does not hold the similar features may result in a failed prediction

  • IP : Ip addresses of the clicks are given
  • App : The app id for marketing
  • Device : The device type id of the user mobile phones
  • OS : The OS version id of the mobile phones
  • Channel : The channel id of different advertisement publishers
  • Hour : Hour which the click took place
  • Day : Day which the click took place

After the prediction process is completed, a csv file will be saved locally containing the prediction results. The results will be shown as 0 and 1 with the relevent click id. In the results column, 0 represents a legitimate click and 1 represents a fraud click.
